# Server Binary Implementation Plan

**Status: SHIPPED** in commit `6393405` (sandbox provider excluded — see `2026-04-24-sandbox-executor.md`).

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `@mcp-conductor/server` — the `mcp-conductor` CLI binary. Reads a single `conductor.json`, instantiates providers and the audit store, wires them to the gateway, and starts the HTTP server with OpenTelemetry init and graceful shutdown. End-to-end integration tests spin up the binary against a real stub MCP upstream and assert a full client round trip.

**Architecture:** Thin composition package. Pulls in `@mcp-conductor/core`, `@mcp-conductor/gateway`, `@mcp-conductor/provider-mcp`, and `@mcp-conductor/provider-sandbox` as dependencies; contains only the `conductor.json` schema, the provider/audit factory, the `main()` wiring, and the CLI entry.

**Tech Stack:** Node 20 LTS, TypeScript, zod. No new runtime deps beyond sibling workspace packages.

**Pre-reqs:** Plans 1-3 complete (`@mcp-conductor/core`, `@mcp-conductor/provider-sandbox`, `@mcp-conductor/provider-mcp`, `@mcp-conductor/gateway` all built and tested).

```typescript
import { z } from "zod";
import { GatewayConfigSchema } from "@mcp-conductor/gateway";

export const McpProviderEntrySchema = z.object({
  type: z.literal("mcp"),
  name: z.string(),
  transport: z.literal("stdio"),
  command: z.string(),
  args: z.array(z.string()).default([]),
  env: z.record(z.string()).default({}),
  initialListTimeoutMs: z.number().int().positive().optional(),
  callTimeoutMs: z.number().int().positive().optional(),
  reconnect: z.object({
    maxAttempts: z.number().int().positive().optional(),
    initialDelayMs: z.number().int().positive().optional(),
    maxDelayMs: z.number().int().positive().optional(),
  }).optional(),
});

export const SandboxProviderEntrySchema = z.object({
  type: z.literal("sandbox"),
  name: z.string().default("sandbox"),
  code: z.object({
    timeoutMs: z.number().int().positive().optional(),
    permissions: z.object({
      net: z.boolean().optional(),
      read: z.array(z.string()).optional(),
      write: z.array(z.string()).optional(),
      env: z.array(z.string()).optional(),
      run: z.boolean().optional(),
    }).optional(),
    denoBin: z.string().optional(),
  }).optional(),
  shell: z.object({
    timeoutMs: z.number().int().positive().optional(),
    allowedCommands: z.array(z.string()).optional(),
    allowedCwds: z.array(z.string()).optional(),
  }).optional(),
});

export const ProviderEntrySchema = z.discriminatedUnion("type", [
  McpProviderEntrySchema,
  SandboxProviderEntrySchema,
]);

export const AuditConfigSchema = z.object({
  type: z.literal("console"),
  bufferSize: z.number().int().positive().optional(),
});

export const TelemetryConfigSchema = z.object({
  serviceName: z.string().default("mcp-conductor"),
  otlpEndpoint: z.string().default(""),
}).default({});

/** Top-level conductor.json shape. Reuses the gateway's users/groups/server schema. */
export const ConductorConfigSchema = z.object({
  providers: z.array(ProviderEntrySchema).min(1),
  audit: AuditConfigSchema,
  telemetry: TelemetryConfigSchema,
}).merge(GatewayConfigSchema).strict();

export type ConductorConfig = z.infer<typeof ConductorConfigSchema>;
export type ProviderEntry = z.infer<typeof ProviderEntrySchema>;
export type AuditConfig = z.infer<typeof AuditConfigSchema>;
```

## Task 5: main() Wiring

**Files:** `packages/server/src/main.ts`, `packages/server/tests/main.test.ts`

`main(configPath, options?)`:
1. Load `conductor.json` via `JsonFileConfigStore<ConductorConfig>`.
2. `initTelemetry(cfg.telemetry.serviceName)`.
3. Build `ProviderRegistry` from `cfg.providers`.
4. `connectAll()` on the registry.
5. Build `AuditStore` from `cfg.audit`.
6. `startGateway({ config, registry, auditStore })`.
7. Return `{ gateway, registry, auditStore, shutdown }`.

Caller is responsible for handling signals (the CLI does that; tests don't).

- [ ] **Step 1:** Implementation

```typescript
import {
  JsonFileConfigStore,
  ProviderRegistry,
  createLogger,
  initTelemetry,
  shutdownTelemetry,
  type AuditStore,
  type Logger,
} from "@mcp-conductor/core";
import { startGateway, type StartGatewayResult } from "@mcp-conductor/gateway";
import { ConductorConfigSchema, type ConductorConfig } from "./conductor-config.js";
import { createAuditStore } from "./audit-factory.js";
import { createProvider } from "./provider-factory.js";

export interface MainOptions {
  manageSignals?: boolean;
  logger?: Logger;
}

export interface MainResult {
  config: ConductorConfig;
  gateway: StartGatewayResult;
  registry: ProviderRegistry;
  auditStore: AuditStore;
  shutdown(): Promise<void>;
}

export async function main(configPath: string, options: MainOptions = {}): Promise<MainResult> {
  const log = options.logger ?? createLogger("server");
  const store = new JsonFileConfigStore({ path: configPath, schema: ConductorConfigSchema });
  const config = await store.load();

  initTelemetry(config.telemetry.serviceName);

  const registry = new ProviderRegistry();
  for (const entry of config.providers) registry.register(createProvider(entry));
  await registry.connectAll();

  const auditStore = createAuditStore(config.audit);

  const gateway = await startGateway({
    config, registry, auditStore, logger: log,
    manageSignals: options.manageSignals ?? false,
  });
  log.info("mcp-conductor ready", { address: gateway.address, providers: registry.names() });

  return {
    config, gateway, registry, auditStore,
    async shutdown() {
      await gateway.close();        // closes HTTP, sessions, providers, audit via registry
      await shutdownTelemetry();
    },
  };
}
```
