# Rework Foundation Implementation Plan

**Status: SHIPPED** in commit `be8f522`. Follow-up `40f0a84` recovered `packages/core/src/data/` files that were silently swallowed by a stray `.gitignore` rule.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate mcp-conductor from a single-package Deno project to a pnpm-managed TypeScript monorepo on Node.js. Ship `@mcp-conductor/core` — the foundation package consumed by every other package: the **`ToolProvider` interface** that abstracts how tools reach the gateway (MCP upstream, OpenAPI, GraphQL, sandbox, future backends), a **JSON-file config loader** (single `conductor.json`), a **console-emitting audit store** with a swappable interface, plus logger, OpenTelemetry init, graceful-shutdown registry, shared types, and error classes.

**Architecture:** pnpm workspaces with `packages/*`. Each package has its own `package.json`, `tsconfig.json`, and tests; the root orchestrates install/build/test. Node.js 20+ runtime, TypeScript compiled via `tsc` for publish and executed via `tsx` in dev.

**Data access is interface-first so we can swap implementations later without touching consumers.** Stage 1 ships `ConsoleAuditStore` (emits audit events as structured JSON lines to stderr, plus a bounded in-memory ring buffer for `queryCalls`) and `JsonFileConfigStore`. Stage 2 will slot in `SqliteAuditStore` / `PostgresAuditStore` / DB-backed config stores without any call-site changes.

**Tool sourcing is provider-based.** The gateway doesn't know or care whether a tool originates from a stdio MCP upstream, an OpenAPI spec, a GraphQL schema, or the sandbox — it calls `provider.listTools()` / `provider.callTool(...)`. Implementations live in sibling packages (`@mcp-conductor/provider-*`). Stage 1 ships MCP and Sandbox; OpenAPI and GraphQL are later plans.

**Tech Stack:** Node.js 20 LTS, TypeScript (NodeNext), pnpm workspaces, tsx, vitest, zod, @opentelemetry/sdk-node.

**Runtime decision (Node vs Bun):** Node wins because `@modelcontextprotocol/sdk`'s `StreamableHTTPServerTransport` expects Node `IncomingMessage`/`ServerResponse` (Bun's Fetch `Response` doesn't match), `@opentelemetry/sdk-node` only fully supports Node, and `child_process.spawn` semantics are what the sandbox needs.

export type LogLevel = "debug" | "info" | "warn" | "error";
const PRIORITY: Record<LogLevel, number> = { debug: 10, info: 20, warn: 30, error: 40 };

export interface Logger {
  debug(msg: string, ctx?: Record<string, unknown>): void;
  info(msg: string, ctx?: Record<string, unknown>): void;
  warn(msg: string, ctx?: Record<string, unknown>): void;
  error(msg: string, ctx?: Record<string, unknown>): void;
  child(scope: string): Logger;
}

function serializeCtx(ctx: Record<string, unknown>): Record<string, unknown> {
  const out: Record<string, unknown> = {};
  for (const [k, v] of Object.entries(ctx)) {
    out[k] = v instanceof Error ? { name: v.name, message: v.message, stack: v.stack } : v;
  }
  return out;
}

function resolveLevel(override?: LogLevel): LogLevel {
  if (override) return override;
  const env = process.env.LOG_LEVEL as LogLevel | undefined;
  return env && PRIORITY[env] !== undefined ? env : "info";
}

export function createLogger(scope: string, levelOverride?: LogLevel): Logger {
  const level = resolveLevel(levelOverride);
  const min = PRIORITY[level];
  function log(lvl: LogLevel, msg: string, ctx?: Record<string, unknown>): void {
    if (PRIORITY[lvl] < min) return;
    console.error(JSON.stringify({
      ts: new Date().toISOString(), level: lvl, scope, msg,
      ...(ctx ? serializeCtx(ctx) : {}),
    }));
  }
  return {
    debug: (m, c) => log("debug", m, c),
    info:  (m, c) => log("info", m, c),
    warn:  (m, c) => log("warn", m, c),
    error: (m, c) => log("error", m, c),
    child: (c) => createLogger(`${scope}.${c}`, level),
  };
}
```

import { NodeSDK } from "@opentelemetry/sdk-node";
import { OTLPTraceExporter } from "@opentelemetry/exporter-trace-otlp-http";
import { resourceFromAttributes } from "@opentelemetry/resources";
import { ATTR_SERVICE_NAME, ATTR_SERVICE_VERSION } from "@opentelemetry/semantic-conventions";
import { trace, metrics, type Tracer, type Meter } from "@opentelemetry/api";

let initialized = false;
let sdk: NodeSDK | null = null;

export function initTelemetry(serviceName = "mcp-conductor", serviceVersion = "0.2.0"): void {
  if (initialized) return;
  initialized = true;
  const url = process.env.OTEL_EXPORTER_OTLP_ENDPOINT;
  if (!url) { console.error("[telemetry] OTEL_EXPORTER_OTLP_ENDPOINT not set — traces disabled"); return; }
  sdk = new NodeSDK({
    resource: resourceFromAttributes({
      [ATTR_SERVICE_NAME]: serviceName,
      [ATTR_SERVICE_VERSION]: serviceVersion,
    }),
    traceExporter: new OTLPTraceExporter({ url: `${url}/v1/traces` }),
  });
  sdk.start();
  console.error(`[telemetry] OTLP exporter → ${url}`);
}

export async function shutdownTelemetry(): Promise<void> {
  if (!sdk) return;
  try { await sdk.shutdown(); }
  finally { sdk = null; initialized = false; }
}

export function getTracer(name: string): Tracer { return trace.getTracer(name); }
export function getMeter(name: string): Meter { return metrics.getMeter(name); }
```

```typescript
import { createLogger, type Logger } from "../logger/index.js";

export interface ShutdownRegistry {
  register(name: string, fn: () => Promise<void> | void): void;
  shutdown(reason: string): Promise<void>;
}

export interface ShutdownOptions {
  registerSignals?: boolean;
  timeoutMs?: number;
  logger?: Pick<Logger, "info" | "error">;
}

export function createShutdownRegistry(options: ShutdownOptions = {}): ShutdownRegistry {
  const registerSignals = options.registerSignals ?? true;
  const timeoutMs = options.timeoutMs ?? 10_000;
  const log = options.logger ?? createLogger("lifecycle");
  const handlers: { name: string; fn: () => Promise<void> | void }[] = [];
  let running: Promise<void> | null = null;

  async function runAll(reason: string): Promise<void> {
    log.info("shutdown", { reason, count: handlers.length });
    for (let i = handlers.length - 1; i >= 0; i--) {
      const h = handlers[i]!;
      try { await h.fn(); log.info("handler done", { name: h.name }); }
      catch (error) { log.error("handler failed", { name: h.name, error }); }
    }
  }

  function shutdown(reason: string): Promise<void> {
    if (running) return running;
    running = (async () => {
      const forced = new Promise<void>((resolve) => {
        setTimeout(() => { log.error("shutdown timeout; forcing", { timeoutMs }); resolve(); }, timeoutMs).unref();
      });
      await Promise.race([runAll(reason), forced]);
    })();
    return running;
  }

  if (registerSignals) {
    const handle = (signal: NodeJS.Signals) => { shutdown(`signal:${signal}`).then(() => process.exit(0)); };
    process.on("SIGINT", handle);
    process.on("SIGTERM", handle);
  }
  return {
    register(name, fn) { handlers.push({ name, fn }); },
    shutdown,
  };
}

import { ProviderError } from "../errors/index.js";
import type { ToolProvider } from "./tool-provider.js";

/**
 * Holds the set of providers by name and exposes lookup. Names must be unique and
 * must not contain `__` (the gateway uses `__` as the tool-namespace separator;
 * allowing it in provider names would make namespace parsing ambiguous).
 */
export class ProviderRegistry {
  private readonly byName = new Map<string, ToolProvider>();

  register(provider: ToolProvider): void {
    if (this.byName.has(provider.name)) {
      throw new ProviderError(`duplicate provider name: ${provider.name}`, provider.name);
    }
    if (provider.name.includes("__")) {
      throw new ProviderError(`provider name must not contain "__": ${provider.name}`, provider.name);
    }
    if (!/^[a-zA-Z0-9][a-zA-Z0-9_-]*$/.test(provider.name)) {
      throw new ProviderError(`invalid provider name: ${provider.name}`, provider.name);
    }
    this.byName.set(provider.name, provider);
  }

  get(name: string): ToolProvider | undefined { return this.byName.get(name); }
  require(name: string): ToolProvider {
    const p = this.byName.get(name);
    if (!p) throw new ProviderError(`unknown provider: ${name}`, name);
    return p;
  }
  names(): string[] { return Array.from(this.byName.keys()); }
  all(): ToolProvider[] { return Array.from(this.byName.values()); }

  async connectAll(): Promise<void> {
    for (const p of this.byName.values()) await p.connect();
  }
  async closeAll(): Promise<void> {
    for (const p of this.byName.values()) {
      try { await p.close(); } catch { /* swallow; caller logs */ }
    }
  }
}

```typescript
const SENSITIVE_KEY_RE =
  /password|secret|token|api[_-]?key|apikey|authorization|auth|credential|private[_-]?key|bearer/i;
const DEFAULT_MAX_BYTES = 4096;

export interface RedactOptions { maxBytes?: number; extraKeys?: string[]; }

function isSensitiveKey(key: string, extra: string[]): boolean {
  if (SENSITIVE_KEY_RE.test(key)) return true;
  const lower = key.toLowerCase();
  return extra.some((k) => k.toLowerCase() === lower);
}
function redactValue(value: unknown, extra: string[]): unknown {
  if (value === null || typeof value !== "object") return value;
  if (Array.isArray(value)) return value.map((v) => redactValue(v, extra));
  const out: Record<string, unknown> = {};
  for (const [k, v] of Object.entries(value as Record<string, unknown>)) {
    out[k] = isSensitiveKey(k, extra) ? "[REDACTED]" : redactValue(v, extra);
  }
  return out;
}
export function redactArgs(args: unknown, options: RedactOptions = {}): string {
  const extra = options.extraKeys ?? [];
  const maxBytes = options.maxBytes ?? DEFAULT_MAX_BYTES;
  const redacted = redactValue(args, extra);
  let json: string;
  try { json = JSON.stringify(redacted); }
  catch { json = '"[UNSERIALIZABLE]"'; }
  if (json.length > maxBytes) json = json.slice(0, maxBytes) + '..."[TRUNCATED]"';
  return json;
}

- [ ] **Step 3:** `packages/core/src/data/console-audit-store.ts`

Design: every `insertCall` emits one JSON line to stderr (operators pick up via whatever logging pipeline they already run) and appends the row to a bounded in-memory ring buffer. `queryCalls` / `count` search the buffer. No file I/O. No background tasks. `flush()` and `close()` are no-ops so consumers can call them unconditionally.

```typescript
import type { AuditCall, AuditQuery, AuditStore } from "./audit-store.js";

function matches(c: AuditCall, q: AuditQuery): boolean {
  if (q.user && c.user !== q.user) return false;
  if (q.provider && c.provider !== q.provider) return false;
  if (q.tool && c.tool !== q.tool) return false;
  if (q.status && c.status !== q.status) return false;
  if (q.since && c.ts < q.since) return false;
  if (q.requestId && c.requestId !== q.requestId) return false;
  return true;
}

export interface ConsoleAuditStoreOptions {
  /** Max rows retained in memory for queryCalls. Oldest evicted. Default 1000. */
  bufferSize?: number;
  /** Writer for audit lines. Defaults to `console.error` so audit goes to stderr alongside logs. */
  writer?: (line: string) => void;
}

export class ConsoleAuditStore implements AuditStore {
  private readonly bufferSize: number;
  private readonly writer: (line: string) => void;
  private readonly buffer: AuditCall[] = [];
  private nextId = 1;

  constructor(options: ConsoleAuditStoreOptions = {}) {
    this.bufferSize = options.bufferSize ?? 1000;
    this.writer = options.writer ?? ((line) => console.error(line));
  }

  async insertCall(call: Omit<AuditCall, "id">): Promise<number> {
    const id = this.nextId++;
    const row: AuditCall = { id, ...call };
    this.writer(JSON.stringify({ kind: "audit", ...row }));
    this.buffer.push(row);
    if (this.buffer.length > this.bufferSize) this.buffer.shift();
    return id;
  }

  async queryCalls(q: AuditQuery = {}): Promise<AuditCall[]> {
    const limit = q.limit ?? 100;
    const matched = this.buffer.filter((c) => matches(c, q));
    matched.sort((a, b) => (a.ts < b.ts ? 1 : a.ts > b.ts ? -1 : 0));
    return matched.slice(0, limit);
  }

  async count(q: AuditQuery = {}): Promise<number> {
    return this.buffer.filter((c) => matches(c, q)).length;
  }

  async flush(): Promise<void> { /* stderr is line-buffered; nothing to flush */ }
  async close(): Promise<void> { /* no resources */ }
}
```

```typescript
import { readFile } from "node:fs/promises";
import type { ZodType } from "zod";
import { ConfigError } from "../errors/index.js";
import type { ConfigStore } from "./config-store.js";

export interface JsonFileConfigStoreOptions<T> {
  path: string;
  schema: ZodType<T>;
}

export class JsonFileConfigStore<T> implements ConfigStore<T> {
  readonly source: string;
  private readonly schema: ZodType<T>;
  private cached: T | null = null;

  constructor(options: JsonFileConfigStoreOptions<T>) {
    this.source = options.path;
    this.schema = options.schema;
  }

  async load(): Promise<T> {
    if (this.cached) return this.cached;
    return this.reload();
  }

  async reload(): Promise<T> {
    let raw: string;
    try { raw = await readFile(this.source, "utf8"); }
    catch (e) { throw new ConfigError(`cannot read config: ${(e as Error).message}`, this.source); }

    let parsed: unknown;
    try { parsed = JSON.parse(raw); }
    catch (e) { throw new ConfigError(`invalid JSON: ${(e as Error).message}`, this.source); }

    const result = this.schema.safeParse(parsed);
    if (!result.success) {
      const issues = result.error.issues.map((i) => `${i.path.join(".") || "<root>"}: ${i.message}`).join("; ");
      throw new ConfigError(`config validation failed: ${issues}`, this.source);
    }
    this.cached = result.data;
    return this.cached;
  }
}

# MCP Conductor — Architecture

Tool gateway for AI agents. Speaks MCP to clients on the front end; aggregates tools from many backends on the back end behind a `ToolProvider` abstraction.

## Runtime

Node.js 20+ (LTS). TypeScript throughout, compiled via `tsc` in each package's `dist/`, executed via `tsx` in dev. No Deno runtime for the main process — Deno is only a subprocess invoked by `@mcp-conductor/provider-sandbox` for permission-bounded code execution.

## Packages

### @mcp-conductor/core (Plan 1)

- `logger/` — structured JSON to stderr, level filtering, child scopes
- `telemetry/` — OpenTelemetry SDK init, tracer/meter factories, shutdown flush (uses `resourceFromAttributes` for OTel 1.30+)
- `lifecycle/` — LIFO shutdown registry wired to SIGINT/SIGTERM with forced-exit timeout
- `errors/` — `ConfigError`, `AuthError`, `ProviderError`, `SandboxError`
- `types/` — `Result<T>`, `Timed<T>`
- **`providers/` — `ToolProvider` interface, `ProviderRegistry`**. Every tool backend (MCP upstream, sandbox, OpenAPI, GraphQL, future) implements this single interface; the gateway talks only to this.
- **`data/` — data-access layer.** `AuditStore` interface + `ConsoleAuditStore` (JSON-line stderr emission + bounded in-memory ring buffer for queries); `ConfigStore<T>` interface + `JsonFileConfigStore<T>` impl with Zod validation; `redactArgs()` for scrubbing sensitive keys. Designed so Stage 2 can ship `SqliteAuditStore` / `PostgresAuditStore` without touching consumers.

### @mcp-conductor/provider-sandbox (Plan 2)

Implements `ToolProvider` with two tools:
- `run_code` — Deno subprocess with per-call permission flags; return value is extracted via an **explicit `returnExpression` option** (no heuristic parsing)
- `run_shell` — Node `child_process.spawn` without shell invocation; commands parsed argv-style, shell metacharacters rejected, optional allowlist

### @mcp-conductor/provider-mcp (Plan 3, together with gateway)

Implements `ToolProvider` by connecting to an upstream MCP server over stdio or SSE via `@modelcontextprotocol/sdk`. Caches tool list, forwards `callTool` with an `AbortSignal`, reconnects with capped exponential backoff on error.

### @mcp-conductor/gateway (Plan 3)

- Hono HTTP app served via `@hono/node-server`. `/mcp` route hands raw Node `IncomingMessage`/`ServerResponse` to `StreamableHTTPServerTransport`.
- Reads from a `ConfigStore<GatewayConfig>` — users (with **sha256-hashed** API keys, compared via `timingSafeEqual`), groups, provider access rules.
- Per-user tool filtering based on group access.
- Namespaced tool names (`{provider}__{tool}`). Provider names containing `__` are rejected at registry time.
- Every tool call: `AuditStore.insertCall({ args: redactArgs(args), requestId })` + OTel span.
- `exportMcpApp(deps)` — mountable on any Node HTTP server.
- Graceful shutdown through the core registry: close HTTP, close sessions, close providers, flush audit, flush OTel.

### @mcp-conductor/server (Plan 4)

The `mcp-conductor` binary.
- Reads `conductor.json` via `JsonFileConfigStore`.
- Instantiates providers per `providers[]` entry (`type: "mcp"` → provider-mcp, `type: "sandbox"` → provider-sandbox).
- Registers providers with `ProviderRegistry`.
- Creates `ConsoleAuditStore` from the `audit` block.
- Starts the gateway HTTP server.
- End-to-end tests spin up a mock stdio MCP upstream and assert a full round trip.

Stage 1 supports `audit.type: "console"` only — every tool call is emitted as a JSON line on stderr (operators ship it with whatever log pipeline they already run: Datadog, Loki, CloudWatch, stdout-to-disk). A bounded in-memory ring buffer (default 1000 entries) backs `queryCalls` for "recent calls" API requests. Stage 2 will add `"sqlite"` and `"postgres"` — same config shape, different `type` value, different `AuditStore` implementation wired automatically.

## Data flow (gateway tool call)

```
MCP client
   │  POST /mcp  Authorization: Bearer <apiKey>
   │  X-Request-Id: <uuid>
   ▼
Node http.createServer (raw req/res for /mcp)
   ▼
Auth: sha256(key) | timingSafeEqual vs stored hashes → user
   ▼
McpServer (per-session, capped by config.server.maxSessions)
   │  tool list filtered by user's group access
   ▼
tool call "{provider}__{tool}" args
   ▼
ProviderRegistry.require(provider).callTool(tool, args, { user, requestId, signal })
   ▼
AuditStore.insertCall({ args: redactArgs(args), status, durationMs, requestId })
OTel span.end()
   ▼
MCP client response
```
